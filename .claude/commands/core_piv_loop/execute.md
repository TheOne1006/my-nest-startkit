# 执行: 根据计划实施

## 待执行计划

读取计划文件：`$ARGUMENTS`

## 执行说明

### 1. 阅读并理解

- 仔细阅读**整个**计划
- 理解所有任务及其依赖关系
- 注意要运行的验证命令
- 审查测试策略

### 2. 按顺序执行任务

对于 "逐步任务 (Step by Step Tasks)" 中的**每个**任务：

#### a. 导航到任务
- 识别所需的文件和操作
- 如果要修改，请阅读现有的相关文件

#### b. 实施任务
- 严格遵循详细规范
- 保持与现有代码模式的一致性 (遵循 NestJS 风格)
- 包含适当的类型提示和文档 (TSDocs, Swagger Decorators)
- 在适当的地方添加结构化日志 (Winston)

#### c. 随时验证
- 每次文件更改后，检查语法
- 确保导入正确
- 验证类型定义是否正确

### 3. 实施测试策略

完成实施任务后：

- 创建计划中指定的所有测试文件 (`*.spec.ts`, `test/*.e2e-spec.ts`)
- 实施所有提到的测试用例
- 遵循概述的测试方法
- 确保测试覆盖边缘情况

### 4. 运行验证命令

按顺序执行计划中的**所有**验证命令：

```bash
# 完全按照计划中的指定运行每个命令
# 例如: npm run lint, npm test, npm run test:e2e
```

如果任何命令失败：
- 修复问题
- 重新运行命令
- 只有当它通过时才继续

### 5. 最终验证

在完成之前：

- ✅ 计划中的所有任务已完成
- ✅ 所有测试已创建并通过
- ✅ 所有验证命令通过
- ✅ 代码遵循项目公约 (`CLAUDE.md`)
- ✅ 文档已按需添加/更新 (Swagger, README)

提供摘要：

### 已完成的任务
- 已完成的所有任务列表
- 已创建的文件（带路径）
- 已修改的文件（带路径）

### 已添加的测试
- 已创建的测试文件
- 已实施的测试用例
- 测试结果

### 验证结果
```bash
# 每个验证命令的输出
```

### 准备提交
- 确认所有更改已完成
- 确认所有验证通过
- 准备好运行 `/commit` 命令

## 注意事项

- 如果遇到计划中未解决的问题，请记录下来
- 如果需要偏离计划，请解释原因
- 如果测试失败，请修复实施直到通过
